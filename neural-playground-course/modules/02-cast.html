Meet the Cast

The four actors in every NeuralPlayground simulation

Every simulation has four players. Once you know their names and roles, you can tell an AI assistant exactly what to change and where — without getting lost in 5,000 lines of code.

Think of it as a stage play. There is a set, an actor, a script consultant, and a director. Each has exactly one job — and they can be swapped out without touching the others.

The Four Players

🏟️
Arena — The set

The virtual environment the agent moves through: a square room, a real-world maze layout, or any geometry you define. It knows about walls, positions, and rewards.

🧠
Agent — The actor

The brain model — a computational theory of how neurons learn to navigate. Weber2018, Stachenfeld2018, or any new model you bring in. Must follow the abstract class contract.

🔬
Experiment — The script consultant

Real rat neural recordings from published papers. Hafting2008, Sargolini2006 — actual spike times and positions from real animals used as scientific ground truth.

🎬
Backend — The director

The orchestration layer that runs the training loop, coordinates the other three components, and logs results. It calls the shots so no one else has to.

Where Each Player Lives in the Code

The folder structure mirrors the four-actor model exactly. Each concern has its own directory — find the right folder and you find the right file.

neuralplayground/main package
agents/ 🧠 Brain models — AgentCore, Weber2018, Stachenfeld2018
arenas/ 🏟️ Virtual environments — Simple2D, Hafting2008Arena, Sargolini2006Arena
experiments/ 🔬 Real rat recordings — Hafting2008Data, Sargolini2006Data
backend/ 🎬 Training orchestration — training_loops, SimulationManager
comparison/ 📐 Scientific metrics — GridScorer
plotting/ 📊 Visualization utilities

Separation of Concerns

This pattern — splitting responsibilities into focused roles — is the most important design principle in software. It is why you can swap Weber2018 for Stachenfeld2018 with one line of code, or drop in a brand-new arena without touching the brain model at all.

Reading the Agent Blueprint

Every brain model must follow the AgentCore contract. Using inheritance, each model gets the shared scaffolding for free and only has to fill in its own logic for act() and update().

Attributes ---------- agent_name : str obs_history: list global_steps: int Methods ------- reset(self): act(self, obs): update(self): save_agent(self, save_path: str, raw_object: bool = True): restore_agent(self, save_path: str): """

Key Terms from This Module

Hover or tap any highlighted term in the text above to see its definition. Here is a quick reference for the concepts introduced in this module.

ABC
abstract class

A blueprint that defines required methods without implementing them — every brain model must fulfill this contract.

↳
inheritance

A new class borrows everything from a parent class and only adds what makes it different — Weber2018 inherits from AgentCore.

SoC
separation of concerns

Each folder does exactly one job — which is why swapping Arena or Agent never requires touching the other.

gym
gymnasium

The standard RL environment interface — NeuralPlayground arenas follow it so any compatible agent can plug straight in.

EHC
EHC

Entorhinal-Hippocampal Circuit — the brain region containing place cells and grid cells that every model in this framework tries to explain.
