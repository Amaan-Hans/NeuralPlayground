Teaching the TEM to Want

How we bolted a reward signal onto an unsupervised brain model — and what changed inside the network

An Agent That Does Not Care

The original TEM is purely unsupervised. It explores, it builds an internal map, it learns to predict what it will see next. But it has no goal. Every location is equally interesting. Every step is equally worth learning from.

Real brains do not work this way. Hippocampal place cells fire more reliably near reward locations. Grid cell firing fields distort around goals. The dopamine system broadcasts a reward-prediction error that scales how strongly any given experience is consolidated into memory. The question is: can we add this to TEM without rebuilding it from scratch?

🎯
The Design Constraint

TEM is a large, carefully tuned unsupervised learner. The goal is to add reward sensitivity on top of it — a thin RL layer — without touching the core generative model or breaking the existing Hebbian memory dynamics. Three things need to change: the observation must carry the reward signal, the learning rate must respond to value, and value must propagate backwards from the reward location.

TD(0): The Simplest Reward Propagator

TD(0) maintains a table V where V[s] estimates the total future reward the agent expects to collect starting from state s. Every time the agent transitions from state s to state s', the estimate is nudged:

TD(0) update rule

error = r + γ · V[s'] − V[s]
V[s] = V[s] + α · error

r = reward at s (1 if reward state, else 0)  |  γ = discount (0.95)  |  α = step-size (0.1)

The critical insight: V values near the reward state are updated first, then over many rollouts the non-zero values propagate outward like ripples — states two steps from reward eventually get V ≈ γ², states three steps get V ≈ γ³, and so on. This is "backwards propagation" through the state space, implemented without any gradient or explicit graph traversal.

CODE — agents/whittington_2020.py · _update_td_values() 
for step_i in range(n_steps - 1):
    for env_i in range(self.batch_size):
        s      = history[step_i][env_i][0]
        s_next = history[step_i + 1][env_i][0]
        r      = 1.0 if s == self.reward_states[env_i] else 0.0
        td_err = r + gamma * self.V_table[env_i, s_next] - self.V_table[env_i, s]
        self.V_table[env_i, s] += alpha * td_err
PLAIN ENGLISH

for step_i in range(n_steps - 1) — Loop over every consecutive pair of timesteps in the 20-step rollout

s = history[step_i][env_i][0] — The current state ID for environment env_i at this step

r = 1.0 if s == self.reward_states[env_i] — Is this the reward location? If yes, r=1. If no, r=0. There is one reward per grid, placed at the centre state.

td_err = r + gamma * V[s_next] - V[s] — The surprise: how much better (or worse) was reality than our prediction? Positive error → V[s] was too low. Negative → too high.

V[s] += alpha * td_err — Nudge the estimate in the right direction by a small step. After thousands of rollouts, V[s] converges to the true discounted return from s.

Where does the reward state live?

Each environment gets one reward state, initialised in _init_td() as n_states // 2 — the centre of the discrete state space. Because TEM trains on 16 environments simultaneously, there are 16 independent value tables and 16 independent reward locations. The agent never teleports to the reward; it must walk there and accumulate experience, just like a real rat.

The Reward Channel: Extending X

The original TEM receives a one-hot observation X of shape (batch, 45). To let the network encode reward proximity, we append one extra dimension: the normalised TD value V(s), scaled so the maximum possible discounted return maps to 1.0.

The 46-dim vector enters the model's compression function f_c(). That function splits it: the first 45 dims go through the original two-hot lookup table (producing a 10-bit code), while the reward dim is appended directly, yielding an 11-dim compressed vector. Every subsequent operation in the model — temporal filtering, memory lookup, place-cell formation — now operates on this 11-dim representation, meaning reward proximity is literally woven into the place-cell pattern at every location.

CODE — whittington_2020_model.py · f_c() 
def f_c(self, decompressed):
    n_reward = self.hyper.get("n_td_reward", 0)
    if n_reward > 0:
        obs_part    = decompressed[:, :-n_reward]
        reward_part = decompressed[:, -n_reward:]
        two_hot = torch.stack(
            [self.hyper["two_hot_table"][i]
             for i in torch.argmax(obs_part, dim=1)], dim=0
        ).to(dtype=decompressed.dtype, device=decompressed.device)
        return torch.cat([two_hot, reward_part], dim=1)
PLAIN ENGLISH

obs_part = decompressed[:, :-n_reward] — Slice off the first 45 columns: the ordinary sensory one-hot observation

reward_part = decompressed[:, -n_reward:] — The last column: the normalised TD value for this state, between 0 and 1

two_hot = torch.stack([two_hot_table[argmax(obs)]...]) — Convert each row's one-hot observation to its 10-bit two-hot code, exactly as before

return torch.cat([two_hot, reward_part], dim=1) — Glue the 10-bit sensory code and 1-bit reward channel together into an 11-dim vector. This is what all downstream memory operations will see.

🔗
Why the Loss Function Also Changed

The TEM loss computes cross-entropy between predicted observations and the true observation label. That label comes from argmax(x). With X now being 46-dim, argmax could accidentally pick index 45 (the reward channel) if its value happened to be the largest. To prevent this, the loss now masks: argmax(x[:, :45]). The model predicts sensory observations only; the reward channel is not a prediction target.

Value-Scaled Learning Rate

Knowing that a location is near reward is useful — but the TEM's unsupervised losses treat all visited states identically. To make the network consolidate reward-proximal experiences more strongly, the learning rate passed to the Adam optimiser is scaled by the current batch's average TD value:

Effective learning rate

eff_lr = base_lr × (1 + td_lr_scale × mean_V_normalised)

base_lr = scheduled rate (decays over training)  |  td_lr_scale = 2.0  |  mean_V_normalised = mean(V_current_states) × (1 − γ)

At the start of training V is everywhere zero, so the effective rate equals the base rate — no change. As values build up near the reward, rollouts that happen to pass through high-value states get a larger gradient step. The multiplier is bounded: at maximum value (1.0 normalised), the rate is tripled (1 + 2.0 × 1.0 = 3). This is intentional — rare, valuable experiences should be consolidated more strongly.

CODE — agents/whittington_2020.py · update() 
curr_values = np.array([
    self.V_table[env_i, history[-1][env_i][0]]
    for env_i in range(self.batch_size)
])
norm_scale    = 1.0 - self.pars.get("td_gamma", 0.95)
avg_norm_value = float(np.mean(curr_values) * norm_scale)
effective_lr  = self.lr * (
    1.0 + self.pars.get("td_lr_scale", 2.0) * avg_norm_value
)
PLAIN ENGLISH

curr_values = np.array([V_table[env_i, last_state] ...]) — Collect the raw TD values for the final state of each environment in the batch — these are the most recently visited states

norm_scale = 1.0 - td_gamma — Multiply by (1 − γ) = 0.05. This scales the maximum possible value (1/(1−γ) ≈ 20) back down to 1.0, giving a normalised signal in [0, 1]

avg_norm_value = mean(curr_values) * norm_scale — Average across the 16 environments to get one scalar signal. High = the batch spent time near reward.

effective_lr = lr * (1 + td_lr_scale * avg_norm_value) — Scale the scheduled learning rate up. At zero value → no change. At full value → 3× the base rate.

What Actually Changed in the Network

Adding one reward dimension to the compressed observation cascades through TEM's architecture. Understanding the cascade requires knowing how the network's internal dimensions are related.

Before

  • n_x = 45  (one-hot observation)
  • n_x_c = 10  (two-hot bits)
  • n_x_f = 10  (per-frequency filtered obs)
  • n_p = n_g_sub × 10  (place cells)
  • sum(n_p) = 400  (total place cells)
  • M shape = (batch, 400, 400)

After (+TD)

  • n_x = 45  (unchanged)
  • n_x_c = 10  (unchanged)
  • n_x_f = 11  ↑ +1 reward dim
  • n_p = n_g_sub × 11  ↑ larger
  • sum(n_p) = 440  ↑ +40 cells
  • M shape = (batch, 440, 440)  ↑ larger memory

The change flows automatically from one line in whittington_2020_parameters.py: n_x_f = n_x_c + n_td_reward. Because every downstream dimension (n_p, W_tile, W_repeat, p_update_mask) is computed from n_x_f, they all resize without further edits. The two-hot table itself is unchanged — it still maps 45 observations to 10-bit codes. The reward channel bypasses that table entirely, joining the representation after compression.

🧩
What the Larger Memory Means

Each place cell is the outer product of a grid-cell code (abstract location) and the compressed sensory+reward vector. With 11-dim sensory+reward instead of 10-dim sensory, the outer product grows proportionally. The memory matrix M is now large enough to bind "where I am" to "what I see AND how much reward is here." In principle, the model can retrieve reward proximity purely from its internal map — without being explicitly told the reward location at inference time.

Three Changes, One Behaviour

The full TD modification touches exactly three files and introduces exactly four new parameters. Here is the complete picture:

whittington_2020_parameters.py

Adds n_td_reward=1, td_gamma=0.95, td_alpha=0.1, td_lr_scale=2.0. Changes n_x_f from n_x_c to n_x_c + n_td_reward. All downstream dimensions cascade automatically.

whittington_2020_model.py

Three surgical edits: f_c() splits the 46-dim input to route obs through two-hot and append the reward channel; b_x is resized from n_x_c to n_x_f[0] to match the wider compressed representation; loss() masks argmax to the first 45 dims so the reward channel does not corrupt prediction targets.

whittington_2020.py (the agent wrapper)

Three new methods: _init_td() builds the value table and places one reward per environment; _update_td_values() runs TD(0) over every rollout; _get_td_signals() returns normalised values to append to X. The existing update() method is extended to call all three and to scale the Adam learning rate by the current batch value.

What you can experiment with

  • td_gamma — Lower it (e.g. 0.5) for a short-horizon agent that only values nearby reward; raise it toward 1.0 for a long-horizon planner.
  • td_alpha — The step-size for value updates. Higher values converge faster but can oscillate.
  • td_lr_scale — Controls how much the model's learning rate amplifies near reward. Set to 0.0 to disable LR scaling while keeping the reward channel in X.
  • reward_states — Currently set to n_states // 2 (arena centre). Change _init_td() to place rewards at corners, multiple locations, or different positions per environment.
  • n_td_reward — Could be extended beyond 1 to encode richer reward signals (e.g. separate channels for different reward magnitudes).
