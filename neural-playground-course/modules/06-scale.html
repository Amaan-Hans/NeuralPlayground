Science at Scale

From one simulation to fifty — automated, logged, and reproducible

A single simulation tells you almost nothing scientifically. You need Weber2018 vs Stachenfeld2018, in multiple arenas, with 5 random seeds each, all results logged and comparable. Without SimulationManager you write 50 loops by hand. With it: three lines of code.

Think of it like a wine competition. Running one simulation is like tasting one wine — interesting, but scientifically meaningless. Good science requires a systematic tasting: same models, five different starting conditions, compared side-by-side with careful notes. SimulationManager is the sommelier who organizes the whole tasting.

N
The combinatorial explosion

N agents × M arenas × K seeds = every combination, automatically. SimulationManager turns a 50-line nested loop into a single configuration object.

✓
Failure-tolerant by design

If one run crashes, the manager logs the error, skips forward, and finishes the rest. Come back, call check_run_status(), and fix only what broke.

📁
Structured output from the start

generate_sim_paths() creates a clean folder hierarchy before the first run — so results are always organized and never overwritten.

Reading the SimulationManager

Five lines control the entire experiment. Here is what each one actually does — and why it matters for reproducibility.

Python from neuralplayground.backend import SimulationManager from neuralplayground.backend.default_simulation import \ weber_in_2d, stachenfeld_in_2d manager = SimulationManager( simulation_list=[weber_in_2d, stachenfeld_in_2d], runs_per_sim=5, manager_id="my_comparison" ) manager.generate_sim_paths() manager.run_all() manager.check_run_status()
In plain English
from ...default_simulation import weber_in_2d, stachenfeld_in_2d
Load pre-configured simulation blueprints — each bundles an agent class, an arena class, a training loop, and all parameters
SimulationManager(simulation_list=[...], runs_per_sim=5, ...)
Set up a manager to run each configuration 5 times — multiple runs average out the randomness of different starting positions
manager.generate_sim_paths()
Create the folder structure on disk where results will be saved — one folder per simulation, per run
manager.run_all()
Execute all 10 simulations (2 configs × 5 runs each) — handles errors gracefully and logs failures separately
manager.check_run_status()
Print a status report: how many completed, how many failed, and what went wrong

The Complete Research Pipeline

Every NeuralPlayground experiment follows the same five steps. Master this sequence and you can describe your entire workflow to an AI assistant with precision — and get usable code back every time.

1
Write Your Agent

Create a class that inherits from AgentCore. Implement act() (return a direction) and update() (adjust weights). The framework handles everything else.

2
Bundle as a SingleSim

Create a SingleSim object that bundles your agent class, an arena class, a training loop function, and a parameters dict into one configuration object.

3
Add to SimulationManager

Pass your SingleSim objects to SimulationManager with runs_per_sim=5. The manager handles the N × M × K combinatorial expansion automatically.

4
Run Everything

Call generate_sim_paths() then run_all(). Failed runs are logged — not fatal. Call check_run_status() to see what completed.

5
Compare Against Experiment Data

Load your saved agents, extract rate maps, and pass them to GridScorer.score() alongside real Experiment data. Numbers — not guesses — tell you whose model is better.

Key Terms from This Module

Hover or tap any highlighted term to see its definition. Here is the quick reference for every concept introduced in this module.

🌱
random seed

Controls the starting randomness of a run — using the same seed gives the same result every time, which is essential for debugging and fair comparison.

♻️
reproducibility

The gold standard of science — if another researcher runs your code and gets a different answer, the result means nothing. Seeded runs and logged parameters make this achievable.

📐
grid score

A quantitative measure of how hexagonally grid-like a neuron's firing pattern is — the key number for comparing models against real rat data.

.pkl
pickle

Python's built-in format for saving any live object to disk — trained agents, environments, and results are stored as .pkl files so nothing is lost between sessions.

💾
serialization

Turning a complex, live Python object into bytes that can be stored and later reconstructed — what happens under the hood every time you call save_agent().

🔴
stack trace

The full error report when something breaks — SimulationManager saves these to log files so you can read them later with show_logs() without re-running anything.
