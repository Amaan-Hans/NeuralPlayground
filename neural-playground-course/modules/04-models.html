How Neurons Learn Space

Two theories, two algorithms — and how grid patterns emerge from nothing

Grid Patterns From Nothing

When a rat explores a room, specific neurons fire ONLY in specific spots — like having a dedicated alarm for each location. Two competing theories explain how this emerges from scratch. Both are in this codebase. Here is how they actually work.

The remarkable thing: nobody programs the hexagonal pattern. It begins with thousands of neurons connected by tiny random numbers. After enough training steps, a perfectly tiled grid appears — spontaneously. Two completely different algorithms produce it through completely different logic.

🧠
What "Emergence" Means Here

Emergence means the grid pattern is not stored anywhere in the code. It is a consequence of the learning rules, the number of neurons, and repeated exposure to a space. Change the learning rate and you get a different pattern. Change the arena shape and the grid adapts. The algorithm produces structure, but the structure itself is never specified.

Weber2018

Pure Hebbian learning. Excitatory and inhibitory neurons compete until spatial structure crystallizes. No explicit reward. No map. Just correlation.

Stachenfeld2018

The Successor Representation. Instead of learning where you are, it learns where you will likely be. Past experience shapes future prediction.

Weber2018: The Musician Metaphor

Imagine two musicians who jam together regularly. The more they play together, the stronger their musical bond — they anticipate each other, finish each other's phrases. In Weber2018, neurons work the same way: if two neurons fire at the same location repeatedly, the connection between them strengthens. That is Hebbian learning in one sentence.

The network has 4,900 excitatory neurons pushing activity up and 900 inhibitory neurons pulling it down. Their competition, repeated thousands of times, carves the space into distinct firing zones.

CODE — agents/weber_2018.py lines 375–419 
def update(self, exc_normalization: bool = True, pos: np.ndarray = None):
    if pos is None:
        pos = self.obs_history[-1]
    if len(pos) == 0:
        pass
    else:
        r_out = self.get_output_rates(pos)
        # Excitatory weights update (eq 2)
        delta_we = self.etaexc * self.get_rates(self.exc_cell_list, pos=pos) * r_out
        # Inhibitory weights update (eq 3)
        delta_wi = (
            self.etainh
            * self.get_rates(self.inh_cell_list, pos=pos)
            * (r_out - self.ro)
        )
        self.we = self.we + delta_we
        self.wi = self.wi + delta_wi
        if exc_normalization:
            self.we = (
                self.init_we_sum / np.sqrt(np.sum(self.we**2) + 1e-8) * self.we
            )
PLAIN ENGLISH

pos = self.obs_history[-1] — Get the most recent position from memory (default: the last visited location)

r_out = self.get_output_rates(pos) — Calculate how strongly the output neuron fires right now at this position

delta_we = self.etaexc * ... * r_out — Excitatory weight change = learning rate × input firing × output firing. Neurons that fire together, wire together.

delta_wi = self.etainh * ... * (r_out - self.ro) — Inhibitory weight change includes a target rate (ro). This prevents the network from firing everywhere ALL the time.

self.we = self.we + delta_we — Apply the update — strengthen the excitatory connections

self.wi = self.wi + delta_wi — Apply the update — strengthen the inhibitory connections

self.we = (self.init_we_sum / np.sqrt(...) * self.we) — Normalize so total weight strength stays constant — this prevents runaway growth

✨
Emergence is Magic

Nobody programs the hexagonal grid pattern into Weber2018. It emerges spontaneously from thousands of steps of simple weight updates. This is one of the most profound results in computational neuroscience.

Stachenfeld2018: The Subway Map Metaphor

Think of a subway map. When you stand at a station, the map does not just show where you ARE — it shows you where you can GO and how many stops away each destination is. Stachenfeld2018 builds exactly this kind of predictive map in the brain: each location encodes not just "I am here" but "here is where I am likely to be in the future."

This is the Successor Representation — a matrix updated by temporal difference learning every time the agent takes a step. The grid-cell-like patterns emerge from eigendecomposition of that prediction matrix.

CODE — agents/stachenfeld_2018.py lines 179–200 
def obs_to_state(self, pos: np.ndarray):
    diff = self.xy_combinations - pos[np.newaxis, ...]
    dist = np.sum(diff**2, axis=1)
    index = np.argmin(dist)
    curr_state = index
    return curr_state
PLAIN ENGLISH

diff = self.xy_combinations - pos[np.newaxis, ...] — For every grid point on the internal map, compute the vector from that point to the agent's current position

dist = np.sum(diff**2, axis=1) — Convert each vector to a scalar distance (squared, to avoid square roots — faster and mathematically equivalent for comparisons)

index = np.argmin(dist) — Find the grid point with the smallest distance — that is the nearest discrete state

return curr_state — Return that index number. A continuous x,y coordinate is now a single integer the SR matrix can use.

Why does this function exist at all?

The arena gives continuous x,y coordinates like [0.312, -0.087]. But the Successor Representation is a matrix indexed by integers — state 0, state 1, state 2. obs_to_state() is the bridge: snap the continuous position to the nearest point on a pre-built grid, return its index. Without this, the continuous world and the discrete math never connect.

Three Models, One Interface

NeuralPlayground ships with three brain models implementing three distinct theories of spatial cognition. Each uses different mathematics, different data structures, even a different framework — but all plug into the same training loop.

🧪

Weber2018

Hebbian plasticity. 4,900 excitatory + 900 inhibitory neurons. Grid cells and place cells emerge from pure correlation learning.

🗺️

Stachenfeld2018

Successor Representation. Builds a predictive map: where will I be in the future? Uses TD learning on a discretized state space.

🔗

Whittington2020

Tolman-Eichenbaum Machine. A graph neural network (PyTorch). Learns relational structure between locations. Requires discrete environments.

🔑
The AgentCore Contract

Three models, three completely different algorithms — but all share the same interface: act() and update(). This is why you can swap them with one line of code.

Decision-making. Given a position, return a direction to move. Any logic is valid — random, learned, hardcoded.

update(self)

Learning. Called after every step. Update internal weights, matrices, or neural network parameters here.
